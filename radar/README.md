# Radar de Oportunidades (Fases 4–5 — não implementado na Sprint 1)

O Radar escaneia watchlists multi-ativo (crypto, tech US, energia, mineração,
commodities/metais, ETFs), roda o backtest+walk-forward default em cada um e os
rankeia por retorno/risco, categorizando em **otimizar / monitorar / ignorar**.

A camada de contexto macro do Radar (quadrantes de inflação, emprego, ciclo
econômico, juros) está especificada em [`../docs/PRD_FUTURE_RADAR.md`](../docs/PRD_FUTURE_RADAR.md).
Esse PRD precisa de pesquisa dedicada **antes** de construir — não chutar quadrantes.

Esta pasta existe agora apenas para fixar a estrutura do PRD. O código entra nas
Fases 4–5, depois que o núcleo BTC estiver validado e as camadas externas (Fase 2)
e o veto IA (Fase 3) existirem.
